Detailed Description

A priority queue (heap), where the top item is the minimum by the sort function.

The user provides a sort function and the top item will be the minimum as per the < relation.

bool parcPriorityQueue_Add ( PARCPriorityQueue *  queue,
void *  data 
)

Add an element to the priority queue, returning true if changed.

A "duplicate" is a data item that compares as equal to another item. The priority queue supports duplicates. It is not stable in regard to the ordering of duplicates. Because it supports duplicates, Add will always return true.

The priority queue is unbounded.

Parameters
[in,out]queueThe queue to modify
[in]dataThe data to add to the queue, which must be comparable and not NULL
Returns
true The data structure was modified by adding the new value
false The data structure was not modified

PARCPriorityQueue* parcPriorityQueue_Create ( PARCPriorityQueueCompareTo *  compare,
PARCPriorityQueueDestroyer *  destroyer 
)

Creates a priority queue with a given sort function.

The sort function defines the ordering of the Priorty Queue. The minimum element will always be the head of the queue.

The destroyer is called on data elements from parcPriorityQueue_Clear() and parcPriorityQueue_Destroy(). You may use } for elements that can be freed by only calling parcMemory_Deallocate.

Parameters
[in]compareDefines the sort order of the priority queue
[in]destroyerCalled for Clear and Destroy operations, may be NULL.
Returns
non-null A pointer to a PARCPriorityQueue

Example:

1 PARCPriorityQueue *q = parcPriorityQueue_Create(parcPriorityQueue_Uint64CompareTo, parcPriorityQueue_ParcFreeDestroyer);
2 uint64_t *objectid = parcMemory_Allocate(sizeof(uint64_t));
3 objectid = 100;
4 
5 // this will use parcPriorityQueue_Uint64CompareTo sort order
6 parcPriorityQueue_Add(q, objectid);
7 
8 // this will use the ParcFreeDestroyer
9 parcPriorityQueue_Destroy(&q);
