int parcStdlibMemory_MemAlign ( void **  pointer,
size_t  alignment,
size_t  size 
)

Allocate aligned memory.

Allocates size bytes of memory such that the allocation's base address is an exact multiple of alignment, and returns the allocation in the value pointed to by pointer.

The requested alignment must be a power of 2 greater than or equal to sizeof(void *).

Memory that is allocated can be used as an argument in subsequent call Reallocate, however Reallocate is not guaranteed to preserve the original alignment.

Parameters
[out]pointerA pointer to a void * pointer that will be set to the address of the allocated memory.
[in]alignmentA power of 2 greater than or equal to sizeof(void *)
[in]sizeThe number of bytes to allocate.
Returns
0 Successful
EINVAL The alignment parameter is not a power of 2 at least as large as sizeof(void *)
ENOMEM Memory allocation error.

Example:

1 {
2  void *allocatedMemory;
3 
4  int failure = parcStdlibMemory_MemAlign(&allocatedMemory, sizeof(void *), 100);
5  if (failure == 0) {
6  parcStdlibMemory_Deallocate(&allocatedMemory);
7  // allocatedMemory is now equal to zero.
8  }
9 }

Return the number of outstanding allocations managed by this allocator.

When you allocate memory, this count goes up by one. When you deallocate, it goes down by one. A well-behaved program will terminate with a call to parcStdlibMemory_Outstanding() returning 0.

void* rpl_realloc ( void *  oldAlloc,
size_t  newSize 
)

Replacement function for realloc(3).

The standard realloc() function tries to change the size of the allocation pointed to by oldAlloc to newSize, and returns oldAlloc. If there is not enough room to enlarge the memory allocation pointed to by oldAlloc, realloc() creates a new allocation, copies as much of the old data pointed to by oldAlloc as will fit to the new allocation, frees the old allocation, and returns a pointer to the allocated memory.

If oldAlloc is NULL, realloc() is identical to a call to malloc() for size bytes. If newSize is zero and oldAlloc is not NULL, a new, minimum sized object is allocated and the original object is freed. When extending a region allocated with calloc(3), realloc(3) does not guarantee that the additional memory is also zero-filled.

If the realloc(3) function is not compatible with the above constraints (i.e., ‘realloc(NULL, 0)’ returns an invalid pointer), then autoconf tools will define realloc to rpl_realloc so that the native realloc is not used in the main project.
